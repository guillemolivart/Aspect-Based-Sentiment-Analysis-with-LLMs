# Qwen3.5-2B Thinking Presence Penalty Comparison

## Purpose

This note compares the first thinking pilot against the same pilot with a
Transformers-side approximation of Qwen's recommended `presence_penalty=1.5`.

The implementation penalizes only tokens already generated by the model, not
tokens from the prompt. This avoids suppressing aspect labels such as
`restaurant_general` and `food_quality`, which appear in the instructions and
must remain easy to generate in the final JSON.

## Interpretation

The failure is not mostly exact token repetition. It is a deliberation problem:
the model keeps re-checking aspect definitions and edge cases instead of
closing `</think>` and emitting the JSON. Penalizing already generated tokens is
not enough to make Qwen3.5-2B terminate thinking reliably on this ABSA prompt.

The next useful step is not to increase `presence_penalty` blindly. A larger
penalty may distort JSON/aspect labels and still fail to solve deliberation. A
better next experiment is a controlled thinking budget with fallback: allow a
small amount of reasoning, abort if `</think>` is not produced by the budget,
and retry no-thinking from the original prompt.
